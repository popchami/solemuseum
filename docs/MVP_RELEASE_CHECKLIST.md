# Kick×Kick MVP Release Checklist

## 目的

Kick×KickをMVPリリース可能な状態まで持っていくための確認リスト。

ブランド・モデルFactory作業には戻らず、MVP本体の安定化とリリース判定を優先する。

---

## 0. 基本方針

- [ ] Kick×Kickとして進める
- [ ] 過去名称の表記を新規追加しない
- [ ] Riverpod構成を維持する
- [ ] Repository構成を維持する
- [ ] SQLite中心で進める
- [ ] 小さな差分で修正する
- [ ] 各変更後に `flutter analyze` を確認する
- [ ] MVPリリースを優先する

---

## 最終メモ

このチェックリストは、リリース可否を判断するためのもの。

チェックが進まない場合は、新機能追加ではなく、起動・analyze・登録導線の修正を優先する。
