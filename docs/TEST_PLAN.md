# Kick×Kick MVP Test Plan

## 目的

Kick×KickのMVPリリース前に、実機で何をどの順番で確認するかを定義する。

この文書は、`docs/MVP_RELEASE_CHECKLIST.md` と `docs/RELEASE_PLAN.md` を実際の確認手順に落とし込むためのもの。

---

## テスト方針

- MVPリリースに必要な主要導線を優先する
- 新機能追加ではなく、起動・保存・表示・編集・削除の確認を優先する
- エラーが出た場合は小さな差分で修正する
- 修正後は必ず `flutter analyze` を確認する
- Riverpod / Repository / SQLite構成を壊さない
- ブランド・モデルFactory作業には戻らない

---

## 0. 事前準備

### 実行コマンド

`app/` ディレクトリで実行する。

```bash
flutter pub get
flutter analyze
flutter run
```

### 合格条件

- `flutter pub get` が通る
- `flutter analyze` の重大エラーが0
- `flutter run` で実機起動できる
- 起動直後にクラッシュしない

---

## 1. 起動テスト

### 手順

1. アプリを起動する
2. Homeを表示する
3. Collectionへ移動する
4. Stickerへ移動する
5. Settingsへ移動する
6. BottomNavigationで各画面を往復する

### 確認項目

- [ ] アプリが起動する
- [ ] Homeが表示される
- [ ] Collectionが表示される
- [ ] Stickerが表示される
- [ ] Settingsが表示される
- [ ] BottomNavigationが動く
- [ ] 画面遷移でクラッシュしない

---

## 2. DB migration version5 テスト

### 手順

1. 既存DBがある状態で起動する
2. 新規インストール状態で起動する
3. 新規登録を行う
4. 登録データを詳細画面で確認する

### 確認項目

- [ ] DB version 5 で起動できる
- [ ] 既存DBからmigrationできる
- [ ] 新規DBでも起動できる
- [ ] `display_title` が保存される
- [ ] `sticker_text` が保存される
- [ ] `status` が保存される
- [ ] `status` の初期値が `new` になる
- [ ] 既存データが壊れない

- [ ] searchKeywords検索できる

- [ ] 自由入力ブランドで保存できる

### 確認項目

---

## 13. リリース不可条件

以下が1つでも残る場合は、Release 1.0へ進まない。

- [ ] 起動できない
- [ ] `flutter analyze` の重大エラーが残る
- [ ] 新規登録できない
- [ ] 編集できない
- [ ] 詳細画面でクラッシュする
- [ ] DB migrationで既存データが壊れる
- [ ] 写真なし登録でクラッシュする
- [ ] Collectionが表示できない
- [ ] 削除できない
