# 1. Sticker Board 上限変更

## 1.1 新しい上限

1つのSticker Boardに配置できるステッカー数を以下へ変更する。

```text
Free: 10枚
Premium: 30枚
```

## 1.2 旧仕様の扱い

旧仕様では以下だった。

```text
Free: 20枚
Premium: 100枚
```

この旧仕様は廃止する。

今後は、必ず以下を正とする。

```text
Free: 10枚
Premium: 30枚
```

## 1.3 上限到達時の動作

上限に達している場合は、新しいステッカーを追加できない。

ユーザーには、分かりやすい案内を表示する。

例:

```text
このボードに置けるステッカー数の上限に達しました。
Freeでは10枚まで配置できます。
```

Premium導線は後日精査する。

# 2. Sticker Board 表示領域

## 2.1 課題

現在、Sticker Boardの表示領域が狭い。

ステッカーを貼る場所として、画面を十分に使えていない。

## 2.2 修正方針

Sticker Boardは、可能な限り画面端に近い位置まで広く表示する。

方針:

- 余白を最小限にする
- 検索バー、上部メニュー、下部ナビゲーションを除いた領域を広く使う
- 端末サイズに応じて自然に拡張する
- ボードの縦横比は崩さない
- ステッカーが画面外に操作しづらくならないようにする

## 2.3 注意

ボードを広げる際、以下を壊さないこと。

- ステッカー位置
- ステッカー回転
- ステッカー拡大縮小
- 選択枠
- ハンドル
- PNG出力

# 3. 閲覧モードの移動

## 3.1 仕様

閲覧モードでも、ステッカーの移動のみ可能とする。

閲覧モードで可能:

- ステッカー移動

閲覧モードで不可:

- 回転
- 拡大
- 縮小
- 複製
- 削除
- 貼り付け
- デザイン編集

## 3.2 理由

Sticker Boardは机の上で自由に配置を楽しむ機能であるため、閲覧中でも軽い位置調整はできるようにする。

ただし、誤操作を防ぐため、移動以外の編集操作は編集モードに限定する。

# 4. 選択枠・ハンドル修正

## 4.1 課題

現在、選択枠やハンドルがステッカーサイズに正しく追従していない。

例:

- ステッカーを大きくしても選択枠が一緒に大きくならない
- 選択枠がステッカー移動に追従しない
- 回転時に枠とステッカーがずれる
- ハンドル位置がステッカー実サイズと合わない

## 4.2 修正仕様

選択枠とハンドルは、常にステッカーと一体で動作する。

必須条件:

- 移動すると選択枠も移動する
- 拡大すると選択枠も拡大する
- 縮小すると選択枠も縮小する
- 回転すると選択枠も回転する
- ハンドルは常に選択枠に追従する

---

# 5. 回転ハンドル

## 5.1 課題

現在、回転ハンドルを操作しても回転できない。

## 5.2 修正仕様

- 回転ハンドルでステッカーを回転できるようにする
- 回転中心はステッカー中心にする
- 移動、拡大縮小と干渉しないようにする
- 回転中も選択枠とハンドルは追従する

---

# 6. 選択時ツールバー

## 6.1 表示位置

ステッカー選択時、選択枠の下にアイコンツールバーを表示する。

## 6.2 表示するアイコン

MVPで表示するアイコン:

- 貼り付け
- 複製
- 削除
- 拡大
- 縮小
- 最前面

## 6.3 表示しないもの

以下はMVPでは不要。

- 最背面

## 6.4 三メニューとの整理

三メニューと選択時ツールバーで、同じ操作を重複させない。

重複している場合は、操作場所を整理する。

原則:

- 選択中のステッカーに対する操作は、選択時ツールバーへ寄せる
- ボード全体の操作は、三メニューへ残す

---

# 7. 三メニュー整理

## 7.1 削除する項目

三メニューから以下を削除する。

```text
貼り付けるステッカーを選択
```

## 7.2 理由

貼り付け操作は、ステッカー選択後のツールバーで扱う。

三メニューに残すと操作が重複し、ユーザーが迷うため削除する。

---

# 8. 長押し動作

## 8.1 編集モード

編集モードでステッカーを長押しした場合、ステッカーデザイン編集へ遷移する。

```text
編集モード
↓
ステッカー長押し
↓
ステッカーデザイン編集
```

## 8.2 閲覧モード

閲覧モードでは、長押しによるデザイン編集は行わない。

閲覧モードでは移動のみ可能。

---

# 9. ステッカーデザイン編集

## 9.1 上部表示

デザイン編集画面上部に、靴詳細の「ステッカーテキスト」部分を抽出して表示する。

この値をステッカーテキストの初期値として使用する。

## 9.2 元データは変更しない

デザイン編集でテキストを変更しても、スニーカー詳細側のデフォルト値は変更しない。

変更内容は、作成・編集対象のステッカーにのみ反映する。

---

# 10. ステッカーテキスト配置

## 10.1 仕様変更

ステッカーテキストは、オレンジフチの外側ではなく、フチの内側に配置する。

理由:

- 一般的なステッカーはフチ内に文字がある
- ステッカー本体と文字が一体に見える
- 机に貼った時の完成度が上がる

## 10.2 編集可能項目

ステッカーテキストは以下を編集可能にする。

- サイズ
- 位置

## 10.3 位置変更

ステッカーテキストは、フチ内を自由に移動できる。

ただし、文字サイズによってフチ外へはみ出す場合は移動範囲を制限する。

必須条件:

- 文字がオレンジフチの外へ出ない
- サイズ変更後も、配置可能範囲を再計算する
- 文字が大きすぎる場合は、移動できる範囲が狭くなる

# 11. ステッカー保存データ最適化

## 11.1 目的

ステッカーの保持データ量を軽減し、画質を落とさず保存する。

目的:

- 保存容量を抑える
- ボード表示を軽くする
- 移動・回転・拡大縮小を重くしない
- 画質を劣化させない

## 11.2 実装方針

必要に応じて以下を検討する。

- 表示用キャッシュ
- サムネイル画像
- 元画像と表示用画像の分離
- 遅延読み込み
- メモリキャッシュ
- PNG保存方式の見直し
- 同一画像の重複保存削減

## 11.3 重要条件

画質を落とさないこと。

容量削減のために、ステッカー本体画像を低解像度化しない。

もし表示用サムネイルを作る場合も、元画像は保持する。

## 11.4 報告必須

実装時は、どの方法で軽量化したかを作業レポートに記載する。
